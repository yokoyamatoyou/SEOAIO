### チェックリスト（各フェーズで使用）

#### エラーチェック（共通）
- 単体テスト: 全件実行、失敗/skipの理由を `PROGRESS_LOG.md` に記録
- スモークテスト（ローカル）: アプリ起動→URL入力→分析→結果表示まで
- ビルド/デプロイ（クラウド）: Cloud Build 成功、Cloud Run 起動、公開URLアクセス

#### デザイナー視点（UI）
- 階層: 見出し/余白/区切り線で情報階層が明確
- 配色: コントラスト/可読性が十分、重要要素が視覚的に強調
- 一貫性: コンポーネント（ボタン/入力/タブ）のスタイルが統一
- レスポンシブ: 縦横リサイズやモバイル幅での崩れがない

#### ユーザー視点（UX）
- 初期導線: 何をすればよいか即時に把握できる（説明/placeholder）
- フィードバック: 進行中/成功/失敗の表示が明確、次アクションが提示される
- エラー時: 原因が分かる説明、再試行/問い合わせの導線
- 成果物: PDF/グラフなどの出力が分かりやすく、再現性がある
